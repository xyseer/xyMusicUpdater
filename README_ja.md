xyMusicUpdater は、**Navidrome** 音楽ライブラリを強化および管理するために設計された、高度にカスタマイズされたフルスタック (Django/React) のコンパニオンアプリケーションです。モダンな Django バックエンドと洗練された React フロントエンドで構築されており、音楽のダウンロード、メタデータタグの取得、オーディオのトリミング、コンピレーションアルバムの統合をインテリジェントに処理します。さらに、Navidrome データベースとシームレスに同期します。

## ✨ 主な機能

*   **自動＆手動検出**: 組み込みのダウンロードエンジンを使用して、ユーザーが定義したプレイリストや個別の URL から直接高品質のオーディオを取得します。
*   **堅牢な重複検出**: 「アクティブ」および「削除済み」のステータスにわたって、ファイル名、ビデオID、および Unicode 正規化されたメタデータを相互参照し、重複ダウンロードを防ぎ帯域幅を節約します。
*   **インテリジェントな自動タグ付け**: ダウンロードしたファイルを **MusicBrainz** や **Apple Music** の API と照合し、カバーアート、アルバム名、アーティストのメタデータを自動的に取得します。
*   **ブラウザ内蔵オーディオエディタ**: Web UI 内から直接、FFmpeg 駆動の「プレビュー後に確定」するワークフローを使用して、オーディオファイルを正確にトリミングできます。
*   **コンピレーションアルバム統合**: 複数のアーティストが参加しているアルバムを自動検出し、ワンクリックで「Various Artists」として統合するツールを提供します。これにより、ライブラリをクリーンに保つことができます。
*   **音響重複スキャナー**: **Chromaprint**（`fpcalc`）を使用してNavidromeライブラリ全体をフィンガープリント処理し、タグ・ファイル名が異なる場合やイントロ/アウトロのオフセットがある場合でも、音響的に同一のトラックをグループ化します。ページネーション付きUIで曲ごとに保持/削除を選択でき、自動削除は一切行いません。
*   **パージ分析と保護**: ストレージの使用状況を分析し、カスタマイズ可能な保持ポリシーに基づいて古いトラックを安全にアーカイブ/削除します。同時に、「監視対象プレイリスト」にある曲は保護されます。
*   **モダンなセキュリティ**: AES-CBC 暗号化されたパスワード送信、起動ごとに動的に生成される `SECRET_KEY`、および厳格な `@api_auth_required` API ファイアウォールを備えています。
*   **Glassmorphism UI**: スムーズな CSS アニメーション、無限スクロールのマーキー、および動的な背景テーマを備えた、完全レスポンシブな多言語（EN/ZH/JA）React フロントエンド。

## 🏗️ アーキテクチャ

xyMusicUpdater は、Docker Compose 経由で Navidrome と並行して動作します。

1.  **バックエンド (Django 5.x / Python 3.12)**: ビジネスロジック、FFmpeg オーディオ処理、SQLite データベース管理を処理し、RESTful API と Server-Sent Events (SSE) ストリームを公開します。
2.  **フロントエンド (React 18 / Vite)**: デスクトップアプリのようなリッチな管理ダッシュボードを提供するシングルページアプリケーション (SPA)。
3.  **Navidrome**: 基盤となるメディアサーバーおよび Subsonic API プロバイダーとして機能します。

*注：xyMusicUpdater は、定期的な再スキャンを待たずに即時のメタデータ同期を実現するために、共有 Docker ボリュームを介して Navidrome の基盤となる `navidrome.db` (SQLite) と直接やり取りします。*

## 🚀 インストールとセットアップ

xyMusicUpdater は Docker Compose 経由でデプロイされるように設計されています。

### 1. 前提条件
*   ホストマシンに Docker と Docker Compose がインストールされていること。

### 2. デプロイ
標準のテンプレート `docker-compose.example.yml` は、Navidrome と xyMusicUpdater を統合します。
このテンプレートを `docker-compose.yml` としてフォルダーにコピーし、ボリュームのマウントパス、ユーザー名、パスワードなどの必須引数に入力します。

Docker でのデプロイを希望しない場合は、`Dockerfile` を参照して、Python と Node.js を使用してネイティブにサービスを開始できます。

### 3. サーバーの起動
```bash
# プロジェクトフォルダにて
docker compose up -d
```
UI にアクセスするには、`http://localhost:4534` にアクセスしてください。

## 🔐 認証

*   **デフォルトの資格情報**: `docker-compose.yml` の `APP_USER` および `APP_PASSWORD` によって制御されます。
*   **セキュリティプロトコル**: アプリケーションは、起動ごとに動的な非永続的なシークレットキーを生成します。これは、**コンテナの再起動をまたいでセッションが保持されない**ことを意味します（サービス再起動後にブラウザを更新する必要がある場合があります）。

## 📁 ディレクトリ構成 (コンテナ内)
*   `/music/temp`: 新しいダウンロードと未処理ファイルのステージングエリア。
*   `/music/permanent`: 保護された/お気に入りのトラックのアーカイブ。
*   `/app/data`: SQLite データベース、カスタム背景、一時的なオーディオプレビュー (`/app/data/previews`) 用の永続ボリューム。
*   `/navidrome_data`: Navidrome のデータベースを含む、直接の読み書きアクセス用の共有ボリューム。

## 🛠️ 開発

フロントエンドアセットをビルドまたは変更するには：
```bash
cd frontend
npm install
npm run build
```

バックエンドをネイティブに実行するには、標準の Django WSGI メソッドまたは開発コマンドを使用してバックエンドを開始してください。Django アプリケーションは、コンパイルされた静的ファイルを `frontend/dist` から自動的に配信します。
