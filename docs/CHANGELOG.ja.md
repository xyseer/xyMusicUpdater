# 変更履歴

---

## [1.3.0]

### 新機能
- **SoundCloud プロバイダー**（[WorldVanquisher](https://github.com/WorldVanquisher) による）— SoundCloud が YouTube と並ぶ一級ダウンロード・検索ソースとして追加。手動ダウンロードおよびディスカバリーで SoundCloud URL を自動検出。検索結果にプロバイダーバッジを表示。
- **yt-dlp 自動更新** — スケジューラーベースのバイナリ自動更新。30 日ごとに GitHub から最新の `yt-dlp` リリースをダウンロードし、期限切れを防止。
- **削除なしアーカイブオプション** — 曲ごとの新しい `no_purge` フラグ。「削除なし」に設定された曲は、パージ時に削除されるのではなく永久アーカイブフォルダーに移動される。
- **ライブラリのワイルドカード検索** — ライブラリタブの検索フィールドが `title`、`artist`、`filename` フィールドにわたるサーバーサイドフリーテキスト一致をサポート。
- **ライブラリのトリムボタン** — ライブラリの各行に直接「トリム」ボタンが追加され、その曲が事前選択された状態で音楽エディタータブに切り替わる。
- **重複グループ内のファイルパス** — 重複パネルの各曲カードにファイルシステムのフルパスが表示されるようになり、削除前に異なるディレクトリのコピーを区別しやすくなった。

### 修正
- **削除パスガード**（`duplicates.py`、`song_views.py`）— 重複パネルとライブラリの両方での削除処理が、`PERMANENT_SAVING_DIR` との比較前にフルパスを解決するようになり、アーカイブ済みファイルの誤削除を防止。
- **useSSE リトライアサーション**（フロントエンドテスト）— `useSSE.test.js` のリトライ遅延とリトライ回数のアサーションを実際の動作（5 秒 / 10 回リトライ）に合わせて修正。
- **Deno ランタイム修正** — yt-dlp の JavaScript ランタイムとして `deno` をイメージにインストール。

## [1.2.1]

### 修正
- **コンピレーション候補** — 任意のカスタムアルバムアーティスト（"Various Artists" だけでなく）にマージされたアルバムが提案から正しく除外されるようになった。常に `false` になっていた論理的に不可能な SQL 条件を修正。
- **コンピレーション UI** — アルバムグループごとに「すべて選択 / すべて解除」トグルを追加。セッション限りの「破棄」とは別の永続的な「永久に無視」ボタン（`/app/data/compilation_ignored.json`）を追加。
- **タグ付けカバーのドラッグ＆ドロップ** — オリジナルカバーが常にドラッグ可能になった。ドロップゾーンが `text/uri-list` 経由のページ画像ドラッグを受け付けるようになった。曲間のカバードラッグがドラッグ時にキャンバスデータ URL をキャプチャし、信頼性の高い同期的な配信を実現。

## [1.2.0]

### 新機能
- **重複検出タブ** — ユーザーが起動するバックグラウンドジョブが `fpcalc`（Chromaprint）で Navidrome ライブラリ全体の指紋を取得。スライディングウィンドウ比較（±40 オフセット ≈ ±5 秒）でイントロ/アウトロの長さが異なる場合でも重複を検出。長さウィンドウ：±30 秒。結果は `/app/data/duplicates.json` に保存（マイグレーション不要）。レビュー UI：曲ごとの保持/削除トグル、グループ却下、ページネーション、自動削除なし。
- **モバイルフレンドリー UI** — `useIsMobile` フック（リサイズ対応）。折りたたみ可能な左サイドバー（ロゴトグル、幅 0 ↔ 200 px）。SongTable がモバイルでカードレイアウトに切り替わる。2 列グリッド → 1 列。TaggingPanel の編集セクションが縦積みに。ライブラリに Navidrome へ移動ボタンを追加。パイプライン実行アイコンをプレイに変更。
- **サブスクリプションごとのキーワードブラックリスト** — `SearchSubscription` モデルに `keyword_blacklist` フィールドを追加（マイグレーション 0002）。ディスカバリージョブがカンマ区切りのパターンに一致するタイトルをスキップ。UI 入力は DiscoveryPanel に。グローバル設定から削除。
- **直接ファイルアップロード** — `POST /api/upload/` が複数ファイルの音声（mp3/flac/m4a/opus/ogg/wav）を受け付ける。`TEMP_FOLDER` に保存し、バックグラウンドスレッドで `register_songs` + 増分 Navidrome 再スキャンを実行。フロントエンド：ManualDownload タブのドラッグ＆ドロップゾーン + ファイルピッカー。
- **スマートディスカバリーソート** — フィルタリング前に候補プールを目標クォータの 3 倍に拡張。候補を `upload_date` 新しい順にソート。ソート後に `max_items` までスライス。キーワードサブスクリプションは `ytsearch{3×amount}:kw` を使用して大きなプールを構築。ディスカバリーは `on_file_ready` コールバックで曲ごとの即時登録を実現。
- **コンピレーションマージのカスタムアルバムアーティスト** — マージ UI がアルバムごとのテキスト入力（デフォルト "Various Artists"）を持つようになり、ハードコードされた値を置き換え。
- **ライブラリの削除・再タグボタン** — ライブラリタブの各行にインライン削除（ファイル + DB レコードを削除）と再タグ（`needs_tagging=true` にマーク）ボタンを追加。

### 修正
- `fix(m3u)` — `latest.m3u` が Django Song テーブルではなく Navidrome の `media_file` テーブルを直接クエリするようになり、ダウンロードソースに関わらずすべてのライブラリトラックが表示される。
- `fix(tagging)` — タグ付けパネルが確認/保存/削除後もスクロール位置を保持。ページ全体のリフレッシュなし。
- `fix(compilation)` — コンピレーションマージ時に URL デコード + マルチストラテジーパス解決（`NAVIDROME_MUSIC_ROOT` 環境変数）でコンテナ間ファイルアクセスに対応。
- `fix(tagging-panel)` — `album_artist` フィールドを復元。「オリジナルを使用」から確認ダイアログを削除。`useCallback` TDZ クラッシュを修正。`notify` prop が App から渡されていない問題を修正。

### 改善
- ジョブ履歴のページネーション（`GET /api/jobs/` に `page` + `page_size` クエリパラメーターを追加）。
- `DUPLICATE_THRESHOLD` が設定で構成可能に（デフォルト 0.80、範囲 0.5–1.0）。
- `DEFAULT_PAGE_SIZE` が JobsPanel と DuplicatesPanel のページネーションも制御するように。
- ディスカバリージョブが `on_file_ready` で曲を即時登録（バッチ全体の完了後ではなく、各曲のダウンロード時にライブラリに表示）。

## [1.1.5]

### 新機能
- **AcoustID オーディオフィンガープリント** — `fpcalc` + AcoustID API 統合によるバックエンド自動タグ付け。AcoustID が完全一致した曲は手動レビューなしで自動確認。
- **サーバーサイドページネーション** — すべての大規模リスト（ライブラリ、タグ付け、コンピレーション、パージ分析、音楽エディター）がサーバーサイドページネーションを使用し、`DEFAULT_PAGE_SIZE` で構成可能。
- **フロントエンド直接タグ検索** — タグ付けタブがブラウザから直接 iTunes と MusicBrainz API を呼び出すことでバックエンドの負荷を軽減。バックエンドの `search_musicbrainz_api` はダウンロード後の自動タグ付けのために残存。
- **Navidrome 増分再スキャン** — 各ディスカバリー/ダウンロードジョブがフルライブラリ再スキャンの代わりに軽量な増分再スキャンをトリガーし、大規模ライブラリの待機時間を大幅に短縮。
- **音楽エディターの刷新** — トリムエディターを波形ビジュアライザー、ハンドルドラッグ UI、プレビュー確認フロー、曲リストのページネーションで再構築。
- **CI/CD パイプライン** — バックエンド pytest、フロントエンド vitest、バージョンタグでのマルチアーキテクチャ Docker プッシュ（`linux/amd64` + `linux/arm64`）のための GitHub Actions ワークフロー。

### 修正
- 一部の環境での API リクエストに影響するクロスオリジン（CORS）問題を修正。
- 動的キー生成と WSGI サーバーワーカー起動間の `SECRET_KEY` 競合を修正。
- リファクタリング中に導入されたホームページのリフレッシュと視覚的なリグレッションを修正。

## [1.0.0] — 初回リリース

- Django 5 + DRF バックエンド。React 18 + Vite フロントエンド。Gunicorn（`--workers 2 --threads 8 --timeout 0`）。
- yt-dlp 統合：URL またはキーワードでのダウンロード、プレイリストサポート、MP3 抽出、ffprobe による整合性チェック。
- MusicBrainz + iTunes 自動タグ付け（保留確認フロー付き）。
- Navidrome 統合：Subsonic API 再スキャン、直接 SQLite 読み書き、`latest.m3u` 生成。
- ストレージクォータ管理：しきい値超過時の古い曲のパージ、プレイリスト保護、永続フォルダーへのアーカイブ。
- AES-CBC 暗号化ログイン。コンテナ起動ごとの非永続 `SECRET_KEY`。
- SSE ライブログストリーム。サブスクリプションごとの APScheduler ディスカバリージョブ。
